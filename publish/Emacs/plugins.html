环境

经过大半年对Emacs的坚持使用,我目前基本确定自己的Emacs使用环境和 初 始配置.根据"一年成为Emacs高手(像神一样使用编辑器)"的建议,我的Emacs 配置基于Steve Purcell的配置https://github.com/purcell/emacs.d. 故而, 本文的第一部分为对于Purcell的配置的分析和注解.而使用环境则基本确定为 OSX系统,不想也没有过多的精力再去折腾Linux系统,毕竟可以折腾的东西实在 太多,还是收收心,有的放矢比较好.

补全,菜单

auto-complete

全局启动auto-complete. 如果本行已经完成缩进,Tab键会启动补全. 补全不会跳出补全buffer?
补全的候选项少于5个的时候,不进行循环(cycle). C模式下不用Tab补全.

ido

辅助加载路径,打开文件

全局使用ido. 使用C-f跳出ido模式 flex匹配
寻找文件的范围不限定当前点 如果当前目录下没有文件名,自动换用合并的工作目录来匹配文件名 ido使用虚拟buffers(自动把历史记录加入候选)
使用smex(模糊匹配函数)来调用函数 idomenu (ido+imenu)

编辑,显示

被动效果

  • whitespace-cleanup-mode 储存文件时自动去掉末尾的空格
  • guide-key 运行函数后提示对应的快捷键
  • highlight-escape-sequences 高亮反义字符
  • page-break-lines 显示优化,把page break显示成横线

选择/跳转

  • expand-region 快速标记当前语义块. 例:光标停留在一个字母上时,快速标记整个词. C-=
  • ace-jump-mode 类似Vim下面EasyMotion的插件.键入搜索词,屏幕上的搜索词会高亮并编号, 通过编号快速跳转(类似我在chrome上使用的vim模拟插件的功能). C-; ace-jump-mode, C-: ace-jump-word-mode. C-; 被flyspell屏蔽.可以用下面方法撤销flyspell的快 捷键. 
    (eval-after-load "flyspell"
  '(define-key flyspell-mode-map (kbd "C-.") nil))
(eval-after-load "flyspell"
  '(define-key flyspell-mode-map (kbd "C-;") nil))
(eval-after-load "flyspell"
  '(define-key flyspell-mode-map (kbd "C-,") nil))
  • multiple-cursors 多光标编辑, C-<, C-> 多选择上一行,下一行. C-c c [rcea] 会被 epy (emacs-for-python)的 C-c c 屏蔽.

编辑

  • unfill fill相反的操作,将所有目标行连成一行
  • browse-kill-ring 从kill-ring中选择内容进行插入,M-x启动
  • move-dup 将行或者矩形上下移动 M-<up>, M-<down>
  • whole-line-or-region 如果没有选择文本, M-w, C-w 默认复制或者删除整行.
  • undo-tree 提供类似vim的undo历史

显示

  • highlight-symbol 高亮符号并在符号间快速导航.高亮所有当前选择的符号出现的地方.高亮 M-s h .. M-s h [lpru] 一系列命令highlight不同的东西. M-n, M-p 选择下一个,上一个符号.

记笔记

记笔记方面有很多方案,比如 emacs wiki mode, markdown, org-mode. 目前我选用的是org-mode,功能强大,复杂,可以用来 GTD. 不过又觉得似乎wiki更利于组织知识点,体现知识点之间的联系.这 个以后有时间再研究.

Org-mode

可以跟日历互动进行项目管理, 支持丰富的格式导出,甚至还支 持运行嵌入的代码. Purcell预装了 org-fstree, org-mac-link, org-mac-iCal 包. 简述包的作用… Purcll配置中设置使用ido进行补全;设置了 org-agenda 包括日记,持续 范围,等等;设置了clock;设置mac快捷键;设置了文学编程语言(haskell和 ocaml被禁止).

我安装了org-jekyll包直接将.org笔记中标记为blog的子树导出为jekyll所 需要的.html文件.

Python

Purell的python配置非常简陋,对于用python写大一些的项目是完全不够的,下面给出 一些补全: Python代码分析 一般来说要用到这么几个东西: pymacs, rope, ropemacs. 这些东西和python以及emacs的关系大致如下:

  1. Pymacs是emacs和python沟通的桥梁.
  2. Rope是python重构的工具
  3. Ropemacs是emacs和rope沟通的桥梁
  4. Ropemacs依赖于pymacs

除此之外,我还看到有人推荐 jedi 这个包,这个包据说在代码补全上 比ropemacs要快,但是对重构的支持不好.不过个人觉得动态语言重构是件很不靠 谱的事情,之所以从eclipse+pydev转向vim/emacs就是发现对于动态语言,依靠重 构还不如加强自身对project的掌握.此外我还试用过PyCharm,目前最好的python IDE之一.结果证明一个结论,动态语言的重构从原理上是不可能的.

语法检查 flymake 是emacs内置的语法检查器; pyflakes 是提供 python语法检查的python包,已通过pip安装;elpy会需要flake8;pep8则提供 python代码书写规范检查.

配置包 网上有很多推荐,我试用的第一个是emacs-for-python (epy).

  • EPY自带/pymacs.el/, rope, ropemacs, ropemode. pymacs是elisp程序,自动加

载 epy 会安装自带的相应的依赖,包括编译pymacs.el,pymacs.py编译 rope,ropemacs,ropemode的python文件.pymacs的.egg-info文件会被加入到 python的外来库文件夹下; 但另外几个不会(可能是自带的版本里省略掉 了.egg-info). 除了pymcas以外,这些包也可以全部直接从pip中安装.对于pymacs,则需要 

pip install -e "git+https://github.com/pinard/Pymacs.git#egg=Pymacs"

但是使用epy任然会编译使用自己带的版本.而且可能会跟从pip中安装的版 本冲突.所以就安心用他的好了,不要强迫症… 快捷键:

  • C-c y duplicate a line
  • C-c c duplicate a line and comment the first line
  • C-c l select a whole line
  • C-o open new line as Vim
  • 输入c (class),Tab插入class的snippet

被动:

  • 输入括号会自动把marked region放入括号中 (使用的是skeleton,但是我发 现electric-pair mode就足够了)

-ELPY 有非常好的 Docs 支持,采用company-mode而不是auto-complete进行补全,本 身拥有很好的订制界面.可以从emacs包管理器中安装.对于python依赖,可以手动安装 flake8等python依赖,也可以在安装好elpy之后运行elpy-config来测试依赖是 否完全,并直接从emacs中一键安装(会调用shell命令,把python包安装到$HOME 下面,相当于pip install -user XXX). 快捷键:

  • C-<arrow> 代码块导航
  • M-<arrow> 代码块移动
  • C-c C-f 项目中找文件
  • C-c C-s 项目中grep
  • M-TAB 补全 M-p,n 选择补全项, C-d,w 补全项文档,源代码
  • M-. M-* 跳转到函数定义,跳回原地
  • C-c C-v 语法检查, C-c C-n,p 下一个,上一个错误
  • C-c C-d 文档
  • C-c C-t 测试
  • C-c C-e 重构符号 C-c C-r 复杂重构

如果要用flycheck代替flymake,使用 

(when (require 'flycheck nil t)
  (setq elpy-modules (delq 'elpy-module-flymake elpy-modules))
  (add-hook 'elpy-mode-hook 'flycheck-mode))

欠缺的

  • 代码折叠 很遗憾,至今没有找到很好的解决方 案.emacs自带的方案基于缩进,理论上应该完美支持python,但是实际上似乎对 于中间有空行的代码块解析有问题.包括org-mode的相关模式也不能理想地折 叠代码.

编程IDE   to_complete

Purcell基本没有配置IDE.下面给出我当前使用的配置 ecb + cedet. 首先, 需要自己下载编译最新的cedet, 

bzr checkout bzr://cedet.bzr.sourceforge.net/bzrroot/cedet/code/trunk cedet
cd cedet
make EMACS=[PATH-TO-EMACS]
cd contrib
make EMACS=[PATH-TO-EMACS]

我从网上直接下载的安装包编译失败,不知为何. 为了方便,我在 .emacs 下建 立了 3rd-party 并把 cedet 直接放在这个文件夹下. 编译好之后,要尽早 载入这个cedet以避免被内置的cedet覆盖.我在purcell的init.el的最开头加入 

(load-file "~/.emacs.d/3rd-party/cedet/cedet-devel-load.el")
(load-file "~/.emacs.d/3rd-party/cedet/contrib/cedet-contrib-load.el")
(semantic-mode 1)
(semantic-load-enable-minimum-features)
(semantic-load-enable-code-helpers)

ecb可以直接从包管理器中安装,目前我没有进一步配置,因为感觉并不是很好用.
